

## Batch B: UX Polish Fixes

After reviewing the codebase, here is the analysis and plan for each item:

### #3 — Purple hover ring clipped between list items in settings

**Root cause:** List items in `CategoriesSection.tsx`, `DocumentCategoriesManager.tsx`, and `CategoriesSection`'s inline lists all use `rounded-lg` on individual items inside a `divide-y` container. The `rounded-lg` creates gaps between rows where the `ring-2` hover effect cannot render continuously, and the `divide-y` borders overlap with rounded corners.

**Fix:** Remove `rounded-lg` from individual list items. Add `first:rounded-t-lg last:rounded-b-lg` so only the top/bottom edges of the list are rounded. Also change `hover:ring-2` to a simpler `hover:bg-[#C4B5FD]/10` approach that works within `divide-y`, or use `relative z-10` on hover to ensure the ring renders above adjacent items.

**Files:**
- `src/components/CategoriesSection.tsx` — lines 312, 476, 566 (3 list patterns)
- `src/components/DocumentCategoriesManager.tsx` — line 181

Change item className from:
```
p-3 bg-white dark:bg-card hover:shadow-md hover:ring-2 hover:ring-[#C4B5FD] hover:shadow-[#C4B5FD]/20 transition-all rounded-lg
```
to:
```
p-3 bg-white dark:bg-card hover:bg-[#C4B5FD]/10 hover:shadow-md hover:ring-2 hover:ring-[#C4B5FD] hover:shadow-[#C4B5FD]/20 transition-all relative hover:z-10 first:rounded-t-lg last:rounded-b-lg
```

### #15 — Document section bar should be white

The `DocumentCategoriesManager.tsx` standalone component wraps content in a `<Card>` with `<CardHeader>` and `<CardTitle>`, which adds a grey-tinted card header. Since this component is also rendered inside the `CategoriesSection` tabs (which already have their own framing), the standalone version's Card wrapper creates visual inconsistency.

**Fix:** In `DocumentCategoriesManager.tsx`, ensure the Card has explicit `bg-white dark:bg-card` on the Card element (line 139), matching how other section cards are styled.

### #18 — Settings sections same size and width

**Fix:** Verify `CategoriesSection` root div uses `p-4` (it does at line 23). No changes needed — sections are already consistent. Will skip this item unless visual inspection reveals a specific discrepancy.
