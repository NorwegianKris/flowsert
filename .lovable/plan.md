

# Refactor Certificate Bundle PDF -- Compact Structure

## Summary
Restructure the certificate bundle PDF to eliminate wasteful separator pages and instead use an overview table on page 1 and compact header overlays on the first page of each certificate document. This significantly reduces page count while maintaining audit readability.

## What Changes

### 1. Replace Cover Page with Overview Page (`addOverviewPage`)

The first page will now contain everything the current cover page has, plus a certificate overview table below the divider.

**Layout (top to bottom):**
- Centered title: CERTIFICATE BUNDLE (font 16, bold)
- Subtitle: FlowSert Workforce Compliance (font 9, grey)
- Metadata (left, font 8, muted grey):
  - Company: [name]
  - Person: [name]
  - Role: [role]
  - Generated: [timestamp]
- Thin divider
- Certificate overview table (using `jspdf-autotable`):
  - Columns: Certificate | Issuer | Issue Date | Expiry Date | Status
  - Sorted alphabetically by certificate name
  - Includes ALL certificates (not just those with documents)
  - Status letters: V / E / X (using `getCertificateStatus` from certificateUtils)
  - Grid theme, light lines, no shading, matching matrix visual style
  - Print-safe Helvetica

### 2. Remove Full Separator Pages

Delete the `addSeparatorPage` function entirely. No more standalone metadata pages between certificates.

### 3. Add Compact Header Overlay on First Page of Each Certificate

For each certificate's **first document page only**, draw a compact header block at the top before placing the document image:

**Header block contents (font 7-8, muted grey, left-aligned):**
- Certificate Name (bold)
- Issuing Authority | Issue Date | Expiry Date (single line, normal weight)
- Thin divider line below

**Height:** approximately 14-16mm total

**How it works technically:**
- On the first page of each certificate, the available content area starts below the header (y offset shifts down by ~16mm)
- The document image is scaled to fit the remaining space below the header
- On subsequent pages of the same multi-page PDF certificate, the full content area is used (no header, just margins + footer)

### 4. Eliminate Blank Pages

The current flow calls `addSeparatorPage` (which does `doc.addPage()`) and then `doc.addPage()` again for each document page. The refactored flow will only call `doc.addPage()` once per document page -- no separate separator pages at all.

### 5. Everything Else Stays

- Footer: centered "Page X of Y", right-aligned "Generated by FlowSert"
- Portrait A4
- Standard Helvetica
- Sequential processing with memory cleanup
- Safety limits (30 certs / 200 pages)
- Active personnel only
- Alphabetical sorting

## Technical Details

### Modified file: `src/lib/mergeCertificatesPdf.ts`

**New imports:**
- `autoTable` from `jspdf-autotable` (already installed)
- `getCertificateStatus` from `@/lib/certificateUtils`

**Removed functions:**
- `addSeparatorPage` -- deleted entirely

**Modified functions:**

`addCoverPage` renamed to `addOverviewPage` with new signature:
```ts
function addOverviewPage(
  doc: jsPDF,
  person: Personnel,
  allCertificates: Certificate[], // all certs, not just those with docs
  companyName?: string,
)
```
- Adds metadata including Role field
- Removes Project field (per spec)
- After the divider, renders the overview table using `autoTable` with columns: Certificate, Issuer, Issue Date, Expiry Date, Status

**New function -- `addCertificateHeader`:**
```ts
function addCertificateHeader(doc: jsPDF, cert: Certificate): number
// Returns the y-position where the document content should start
```
- Draws certificate name (bold, font 9) at top margin
- Draws issuer, issue date, expiry date on one line below (font 7, grey)
- Draws thin divider
- Returns y offset (approximately MARGIN + 16)

**Modified main loop in `generateCertificateBundlePdf`:**
- Remove `addSeparatorPage(doc, cert)` call
- For PDF rendering: on pageNum === 1, call `addCertificateHeader` and adjust the available content height; on pageNum > 1, use full content area
- For image rendering: call `addCertificateHeader` on the single page and adjust available height accordingly
- Content height for first page: `CONTENT_HEIGHT - HEADER_HEIGHT` (approximately 16mm less)
- Content is vertically centered within the remaining available space

**Constants:**
- Add `CERT_HEADER_HEIGHT = 16` (mm) for the compact overlay header

### No other files change
The interface (`CertificateBundleOptions`, `CertificateBundleResult`) remains identical. The callers in `ExternalSharingDialog.tsx` and `ShareProjectDialog.tsx` require no updates.
