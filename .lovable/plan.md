
# Personnel Competence Matrix PDF Export

## Overview
Replace the current "Personnel & Certificates" PDF export (which lists certificates per person vertically) with an industry-standard competence matrix format. The matrix uses rows for personnel, columns for certificate types, and symbols for status -- matching offshore/maritime audit conventions.

## What Changes

The `generatePersonnelCertificatesPdf` function in **both** `ShareProjectDialog.tsx` and `ExternalSharingDialog.tsx` will be replaced with a new matrix-style generator.

---

## PDF Layout

### Page Setup
- **Orientation**: Landscape (A4) to maximize column space
- **Margins**: 14mm left/right, 20mm top

### Header Block (every page)
```text
Personnel Competence Matrix

Project: Reparasjon 123              Company: Techno Dive AS
Location: Haugesund                  Generated: 27 Jan 2026 · 08:31
                                     Generated by FlowSert
```

For the global External Sharing dialog (no project context), the header shows "Personnel Overview" instead of a project name.

### Matrix Table
```text
| Personnel          | Role        | Medical | HUET | Diving | Driver's Lic |
|--------------------+-------------+---------+------+--------+--------------|
| Kristian Utseth    | Rigger      | ⚠       | ✖    | ✔      | ✖            |
| Pichet Poonsawat   | Rigger      | —       | —    | —      | —            |
| Oli Pedersen       | Dive Sup.   | ⚠       | —    | ✔      | —            |
```

**Column generation logic:**
1. Collect all unique certificate names/types across the selected personnel
2. Sort alphabetically
3. Each becomes a column header
4. Only certificates present among the personnel are shown (no empty columns)

**Cell logic:**
- Look up each person's certificate matching the column type
- Apply `getCertificateStatus()` to determine the symbol:
  - `valid` --> checkmark symbol
  - `expiring` --> warning symbol
  - `expired` --> cross symbol
  - Not held --> dash symbol

### Legend (bottom of last page)
```text
✔ Valid    ⚠ Expiring within warning period    ✖ Expired    — Not held / not applicable
```

### Footer (every page)
```text
Generated on 27 Jan 2026 08:31 | Page 1 of 2 | Generated by FlowSert
```

## Technical Implementation

### New utility function: `generateCompetenceMatrixPdf`

Create a shared function in `src/lib/competenceMatrixPdf.ts` so both dialogs can use it.

**Parameters:**
```typescript
interface CompetenceMatrixOptions {
  personnel: Personnel[];
  projectName?: string;       // undefined = "Personnel Overview"
  companyName?: string;       // from useBusinessInfo
  location?: string;          // project or company location
}
```

**Logic steps:**
1. Create landscape jsPDF: `new jsPDF('l')`
2. Render header block with project/company info
3. Collect unique certificate types from all personnel
4. Build matrix data: one row per person, one column per cert type, symbol per cell
5. Render using `autoTable` with:
   - `showHead: 'everyPage'` for repeated headers
   - `theme: 'grid'` for clean bordered look (audit-friendly)
   - Monochrome styling (dark header, no blue -- black/white printable)
   - Center-aligned symbol columns
   - Left-aligned name/role columns
6. Add legend below the table
7. Add footer on every page

**Table styling:**
```typescript
headStyles: {
  fillColor: [50, 50, 50],    // Dark gray (not blue -- B&W friendly)
  textColor: [255, 255, 255],
  fontSize: 7,
  fontStyle: 'bold',
  halign: 'center'
}
bodyStyles: {
  fontSize: 8,
  halign: 'center'
}
columnStyles: {
  0: { halign: 'left', cellWidth: 40 },  // Name
  1: { halign: 'left', cellWidth: 25 },  // Role
}
```

### Updates to ShareProjectDialog.tsx
- Import `generateCompetenceMatrixPdf` from the new utility
- Replace `generatePersonnelCertificatesPdf()` call with the new function
- Pass `project.name`, `project.location`, and business name

### Updates to ExternalSharingDialog.tsx
- Import `generateCompetenceMatrixPdf` from the new utility
- Replace `generatePersonnelCertificatesPdf()` call with the new function
- Pass business name from context; project name if a project is selected

### Business name access
- `ShareProjectDialog` will need `businessName?: string` as a new prop (passed from parent)
- `ExternalSharingDialog` already has access to personnel; will also need `businessName?: string`
